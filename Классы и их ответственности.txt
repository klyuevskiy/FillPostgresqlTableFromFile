Main, метод main
    -подключается к БД, вызывает методы нужных классов, получает от них результат(если есть).
    Проверяет действия на наличие ошибок
FileReader
    -взаимодействует с файлом, вроде бы тоже самое предоставляет как Buffered Reader,
    но как и остальные классы в программе обрабатывает Exeptions и сохраняет их к себе
DataBaseArgumentsParser, PostgresqlParser
    -так как setString не приводит к нужному типу сам, то нужно как-то приводить к нужному типу
    В каждой СУБД могут быть свои типы. Получается интерфейс предоствялет метод для парсинга
    PostresqlParser уже получается реализация для моей задачи
TableAnalizator
    - нужен для получения/анализа данных, которые верны в целом для таблицы
    ! Получение запроса для вставки зависит только от имени таблицы
ColumnsAnalizator
    - нужен для получения/анализа данных, которые верны для определнных столбцов определённой таблицы
    - найти соответствия между вводимыми столбцами и имеющимеся
    - получить типы данных нужных столбцов
dataBaseInserter
    -должен осуществлять непосредственную вставку данных в таблицу
    - вставляет данные из файла
